Mobbex Ordenes de Pago

URL

https://api.mobbex.com/p/payment_order

Cabeceras (Headers)

  • x-api-key: Clave API de su aplicación
  • x-access-token: Token de Acceso a la entidad para la cual desea realizar la operación.
  • Content-Type: application/json

JSON Body

Requeridos

  • total: Monto de la operación. Formato XXX.xx donde el PUNTO es el separador decimal.
  • description: Descripción de la operación que será mostrada en el Checkout.

Opcionales

  • email: El email al cual será enviada la orden de pago generada. Si no se provee se usa el teléfono o se retorna la URL
  • phone: Número de Celular al cual será enviada la Orden Generada. Si no se provee se usa email o se retorna la URL. El número debe ingresarse sin 0 y sin 15.
  • reference: Factura, Recibo o referencia del pago a Realizar. Puede ser un identificador de un sistema externo para seguimiento.
  • due: Fecha de vencimiento de la Orden. Formato: day, month, year. Ej: due: { day: 1, month: 2. year: 19 }
  • secondDue: Segundo vencimiento de la Orden. Formato: days, surcharge, días al segundo vencimiento y recargo adicional. Ej: secondDue: { days: 10, surcharge: 100 }
  • actions: Acciones mostradas en la descripción del pago a realizar. Este parámetro es un array y puede contener 1 o varias. Ej. "actions": [{ "icon": "attachment", "title": "Factura", "url": "https://mobbex.com/mifactura/123" }]
  • return_url: URL a la cual será enviado el usuario al finalizar el pago.
  • webhook: URL a la cual será informado el pago mediante WebHooks ( POST )
  • items: Elementos que serán mostrados en el checkout como parte de la descripción del pago. Las imágenes deben soportar HTTPS, de lo contrario no serán mostradas.
{
    "items": [
        {
            "image": "https://www.mobbex.com/wp-content/uploads/2019/03/web_logo.png",
            "quantity": 2,
            "description": "Mi Producto",
            "total": 50
        }
    ]
}
  • options: Otras opciones.
    • El largo máximo para el parámetro smsMessage es 140 caractéres.
  "options": {
        "smsMessage": "Hola esta es tu orden de pago: "
    }
  • sources: Permite la limitación de los medios de pago aceptados. Para realizar dicha limitación se debe enviar un array de medios a aceptar. Ej.
{
    "sources": ["visa", "mastercard"]
}
  • installments: El checkout permite la limitación de los Planes Activos. Para realizar dicha limitación se debe enviar un array de referencias del Plan. Existen 2 formas de Limitar Planes:
  1. Limitar los Planes a las elegidas en el Array:
{
    "installments": ["1", "3", "ahora_12"]
}
  1. Ocultar los Planes a las elegidas en el Array agregando un menos antes de la referencia del Plan:
{
    "installments": ["-ahora_12", "-ahora_18"]
}

Para conocer las opciones disponibles de medios de pago sources y planes installments ver Métodos de pago y cuotas

Para el caso especial de los Planes Ahora ( Argentina ) las referencias siempre serán fijas:

  • ahora_3
  • ahora_6
  • ahora_12
  • ahora_18
  • customerForm: Permite utilizar un formulario diseñado desde Mobbex. Para configurar este campo anteponer custom: al ID del Formulario generado. Ej "custom:45uZCRY09".
  • test: Activa el modo de Pruebas.

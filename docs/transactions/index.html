Transacciones

La URL de transacciones listará todas las transacciones generadas para el pago solicitado mediante la referencia junto con sus estados.

URL: https://api.mobbex.com/2.0/transactions/coupons/{reference}

Parametros (GET Params)

Requeridos

  • reference Referencia de la factura o pago solicitado.

Ejemplos

Bash
JavaScript
PHP

  --url https://api.mobbex.com/2.0/transactions/coupons/mi_referencia_0101 \
  --header 'cache-control: no-cache' \
  --header 'x-api-key: zJ8LFTBX6Ba8D611e9io13fDZAwj0QmKO1Hn1yIj' \
  --header 'x-access-token: d31f0721-2f85-44e7-bcc6-15e19d1a53cc'
var xhr = new XMLHttpRequest();
xhr.withCredentials = true;
xhr.addEventListener("readystatechange", function () {
    if (this.readyState === 4) {
        console.log(this.responseText);
    }
});
xhr.open(
    "GET",
    "https://api.mobbex.com/2.0/transactions/coupons/mi_referencia_0101"
);
xhr.setRequestHeader("x-api-key", "zJ8LFTBX6Ba8D611e9io13fDZAwj0QmKO1Hn1yIj");
xhr.setRequestHeader("x-access-token", "d31f0721-2f85-44e7-bcc6-15e19d1a53cc");
xhr.setRequestHeader("cache-control", "no-cache");
xhr.send();

Consulta de Estado de Transacción

Listado y Búsqueda de Transacciones

Mediante la siguiente API se puede obtener un listado de operaciones y ejecutar diferentes filtros para encontrar las operaciones buscadas en Mobbex.

URL: https://api.mobbex.com/p/operations?page={page}&limit={limit}

Requerido

  • API Key
  • Access Token

Metodos Admitidos

  • GET
  • POST: "content-type: application/json"

Parámetros en URL

  • page: Número de página para paginación
  • limit: Límite de transacciones a devolver. Default: 15. Máximo: 50

Parámetros de Búsqueda admitidos

  • status: Estado de la transacción. Ver Documentación de Estados
  • currency: Moneda de la Transacción: Admitidos: ARS, TEST.
  • created_from: Fecha de Creación de la Operación. Desde cuando buscar. Fecha como String o Timestamp. (Sólo GET)
  • created_to: Fecha de Creación de la Operación. Hasta cuando buscar. Fecha como String o Timestamp. (Sólo GET)
  • created: Fecha de Creación de la Operación. Sólo para Método POST, permite generar un objeto como el siguiente
{
  "from: {
    "day": 10,
    "month": 11,
    "year": 2019
  },
  "to": {
    "day": 11,
    "month": 11,
    "year": 2019
  }
}
  • text: Buscar texto en las operaciones.
  • reference: Referencia asociada a la Operación.
  • test: Permite obtener sólo operaciones de prueba.
  • context: Permite filtrar el contexto de la operación. String separado por coma sin espacios en GET o array de Strings en POST.

Contextos

  • Checkout: plugin.value.checkout:web ,plugin.value.checkout:button
  • Links de Venta: plugin.value.commerce:buy
  • Códigos de Barra de Pago: plugin.value.payment_code:generate
  • Órden de pago: plugin.value.payment_order:web
  • MultiPOS: plugin.value.pos:web, plugin.value.pos:web.mobile, plugin.value.pos:mobile
  • Suscripciones: plugin.value.subscriptions:subscribe, plugin.value.subscriptions:exec

Devolución / Anulación

Mediante este llamado a nuestro API podrás procesar la devolución o anulación de una compra en estado Pago. Es importante aclarar que no todos los medios de pago soportan anulación o devolución del monto abonado.

GET: https://api.mobbex.com/p/operations/{ID}/refund

Parametros (GET)

Requeridos

  • ID: ID de la Operación que se intenta devolver.

Devolución Parcial

Mediante este llamado a nuestro API podrás procesar la devolución o anulación de una compra en estado Pago. Es importante aclarar que no todos los medios de pago soportan anulación o devolución del monto abonado.

POST: https://api.mobbex.com/p/operations/{ID}/refund

Parametros (GET)

Requeridos

  • ID: ID de la Operación que se intenta devolver.

Parametros (POST)

Requeridos

  • total: Monto que se devolverá

Resultado

  • result Resultado de la llamada realizada. Siendo true cuando la operación se anula correctamente y false cuando se produce un fallo.
  • status_message Mensaje de resultado de la llamada o mensaje de la red.

Código QR de Transacción

El código QR generado por Mobbex en las transacciones: Email y Tickets puede ser utilizado para obtener fácilmente el código de la misma y mediante nuestra API consultar el detalle de la Transacción.

Utilización del Código QR.

  1. Leer el código QR utilizando un lector de códigos estandar
  2. Separar el código utilizando el caracter: :
  3. El primer segmento del QR separado será el ID de la transacción
  4. Utilizando el ID de la transacción Consultar el Estado de la misma en nuestro API.

Ejemplo de Código QR

MYpmTQlVC:kYSmcGx1Z2lusHBsdWdpbi52YWx1ZS5wb3OjdWlkrm1vYmJleC53ZWIucG9zpGRhdGHZMHsidXNlciI6IjU3NmQyOTEzYzM0YzY0MTIwMDI1OTU2YSIsInR5cGUiOiJ3ZWIifaNfaWSCqV9ic29udHlwZahPYmplY3RJRKJpZMQMXP3Z3uPhOxJEzKZ5

Código Separado

  • MYpmTQlVC: Código de Operación
  • kYSmcGx1Z2lusHBsdWdpbi52YWx1ZS5wb3OjdWlkrm1vYmJleC53ZWIucG9zpGRhdGHZMHsidXNlciI6IjU3NmQyOTEzYzM0YzY0MTIwMDI1OTU2YSIsInR5cGUiOiJ3ZWIifaNfaWSCqV9ic29udHlwZahPYmplY3RJRKJpZMQMXP3Z3uPhOxJEzKZ5: Datos internos de la Operación

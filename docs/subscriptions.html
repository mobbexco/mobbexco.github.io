Suscripciones

URL Base

https://api.mobbex.com/p/subscriptions

Cabeceras (Headers)

Para el módulo suscripciones TODAS las llamadas requieren el envío de las Cabeceras de Autenticación descriptas a continuación.

  • x-api-key: Clave API de su aplicación
  • x-access-token: Token de Acceso a la entidad para la cual desea realizar la operación.
  • Content-Type: application/json

Creación/Edición

URL y Método:

POST https://api.mobbex.com/p/subscriptions/{id}?

Parámetros de la URL

  • id: Identificador de la suscripción. Sólo para edición de una suscripción, no enviar para crear.

JSON Body ( POST )

Requeridos

  • total: Monto de la operación. Formato XXX.xx donde el PUNTO es el separador decimal.
  • setupFee: Monto inicial diferente al cobrado cada período. Sólo se cobrará una vez. Formato XXX.xx donde el PUNTO es el separador decimal. Sólo disponibles en suscripciones Dinámicas.
  • currency: Moneda de la operación. Sólo admite ARS.
  • type: Valores posibles: dynamic o manual. Las suscripciones de tipo manual deberán procesarse externamente llamando al servicio, no se tendrá en cuenta intervalo en este tipo.
  • name: Nombre de la suscripción.
  • description: Descripción de la operación que será mostrada en el Checkout.
  • interval: Intervalo de Cobro de la suscripción. Sólo puede editarse si la suscripción no posee suscriptores, de lo contrario arroja error. Valores Admitidos: 7d, 15d, 1m, 2m, 3m, 6m, 1y. Donde "d" es Dias, "m" es Meses y "y" es Años. Este parámetro NO aplica a suscripciones manuales.
  • trial: Cantidadd de Periodos de Prueba. Sólo para suscriciones dinámicas.
  • limit: Cantidad de Periodos que se ejecutarán. Colocar 0 para anular los periodos y cobrar ilimitadamente. Colocar un número mayor a 0 para limitar los periodos. Este número siempre debe ser un entero.
  • webhook: URL en la que se recibirán las notificaciones via POST. Debe ser HTTPs.
  • reference: Referencia que identifique a la Suscripción.
  • return_url: URL a la que se enviará al usuario una vez suscripto. Debe ser HTTPs.
  • features: Permite activar o desactivar características internas de Suscripciones. Este campo debe enviarse como array y es opcional. Ej: "features": ["accept_no_funds", "validate_id"]. Opciones:
    • validate_id: Validate ID controlará el DNI del suscriptor contra el Tarjeta-Habiente rechazando la operación en caso de no coincidir.
    • accept_no_funds: Accept No Funds aceptará los casos donde la tarjeta no posea fondos como un caso válido de tokenización.
    • no_email: Desactivará los emails de aviso al Suscriptor pero no los emails de aviso al comercio.
    • no_cc: Desactivará la cuenta corriente, por lo tanto si el cliente posee deuda no intentará recobrarla automáticamente.
    • no_retry: Desactivará el reintento de cobro automático. Si un cobro falla deberá ejecutarlo manualmente.
  • sources: El checkout permite la limitación de los medios de pago aceptados. Para realizar dicha limitación se debe enviar un array de medios a aceptar. Ej.
  • split: Ver Sección Modalidad Split
  • Personalización: Le permite personalizar su Experiencia de Cobro: Ir a la Documentación
{
    "sources": ["visa", "mastercard"]
}

Para conocer las opciones disponibles de medios de pago sources ver Métodos de pago y cuotas. Por el momento para las suscripciones solo pueden configurarse pagos con tarjeta, proximamente se incorporarán más métodos de pago.

Funcionamiento

  1. Crear una nueva suscripción.
  2. Guardar el ID de la Suscripción y el campo "shorten_url"
  3. Para crear un nuevo suscriptor redirigir al usuario a "shorten_url"

Ejecutar cobros manuales masivamente

Tenga en cuenta que esta API Solamente retornará que el proceso fue aceptado exitosamente. La ejecución se realizará de fondo y se notificará via Webhooks.

URL y Método:

GET https://api.mobbex.com/p/subscriptions/{id}/action/execute

Parámetros de la URL

  • id: ID de la Suscripción

JSON Body ( POST )

Importante: Array de Objetos de Tipo JSON

Requeridos

  • sid: Identificador del Suscriptor.
  • total: Monto de la operación. Formato XXX.xx donde el PUNTO es el separador decimal.
  • reference: Referencia que identifique el cobro al suscriptor.
  • description: Permite personalizar la descripción del Cobro.
  • test: Marca la ejecución como Prueba. Valores aceptados: true / false. Defecto: false

Crear Suscriptor

URL y Método:

POST https://api.mobbex.com/p/subscriptions/{id}/subscriber

Parámetros de la URL

  • id: ID de la Suscripción

JSON Body ( POST )

  • customer: Objecto con los datos del Cliente.
  • customer.email: Email del Cliente. Este parametro es Necesario.
  • customer.identification: DNI del Cliente
  • customer.name: Nombre del Cliente
  • customer.phone: Teléfono del CLiente
  • customer.{field}: Puede colocar información sobre su cliente en este objeto. El objeto Customer no debe superar los 20kb.
  • reference: Referencia que identifique al suscriptor.
  • test: Marca al suscriptor como Prueba. Esto no puede cambiarse. Un suscriptor de prueba nunca podrá dejar de serlo. Valores aceptados: true / false. Defecto: false
  • startDate: Objecto con la fecha de inicio.
  • startDate.day: Dia de inicio de la suscripción
  • startDate.month: Mes de inicio de la suscripción.
  • startDate.year: Año de inicio de la suscripción ( Opcional ). Si no se provee se tomará el año corriente.

Ejemplo

{
    "customer": {
        "identification": "32321321",
        "email": "demo@mobbex.com",
        "name": "Demo User"
    },
    "startDate": {
        "day": 15,
        "month": 5
    },
    "reference": "demo_user_321"
}

Respuesta

{
    "result": true,
    "data": {
        "uid": "Mw57pJPU~",
        "reference": "demo_user_321",
        "sourceUrl": "https://mobbex.com/p/subscriptions/qk3lVkjm0/subscriber/Mw57pJPU~/source",
        "subscriberUrl": "https://mobbex.com/p/subscriptions/qk3lVkjm0/subscriber/Mw57pJPU~",
        "subscription": {
            "uid": "qk3lVkjm0"
        }
    }
}
  • sourceUrl: URL a la que debe enviar al usuario a añadir la Tarjeta
  • subscriberUrl: URL de Control del Suscriptor.

Cambiar Agenda del Suscriptor

Esta acción re-generará toda la agenda del suscriptor basado en la fecha de inicio indicada.

URL y Método:

POST https://api.mobbex.com/p/subscriptions/{id}/subscriber/{sid}/action/reschedule

Parámetros de la URL

  • id: ID de la Suscripción
  • sid: ID del Suscriptor

JSON Body ( POST )

  • startDate: Objecto con la fecha de inicio.
  • startDate.day: Dia de inicio de la suscripción
  • startDate.month: Mes de inicio de la suscripción.

Ejemplo Body

{
    "startDate": {
        "day": 15,
        "month": 5
    }
}

Mover Suscriptor a otra Suscripción

Tenga en cuenta que sólo se podrá mover un suscriptor a otra suscripción dentro de la misma entidad. Durante el proceso de mover al suscriptor la agenda del mismo será recalculada cuando la suscripción sea "dynamic"

URL y Método:

POST https://api.mobbex.com/p/subscriptions/{id}/subscriber/{sid}/action/move

Parámetros de la URL

  • id: ID de la Suscripción
  • sid: ID del Suscriptor

JSON Body ( POST )

  • sid: ID de la nueva suscripción

Agendar un Cobro Futuro

Mediante esta API usted podrá programar un cobro que se realizará automáticamente en el día que la programó.

URL y Método:

POST https://api.mobbex.com/p/subscriptions/{id}/subscriber/{sid}/action/schedule

Parámetros de la URL

  • id: ID de la Suscripción
  • sid: ID del Suscriptor

JSON Body ( POST )

  • date: Objecto con la fecha de inicio.
  • date.day: Dia a Agendar.
  • date.month: Mes a Agendar.
  • date.year: Año a Agendar.
  • total: Total específico a cobrar al suscriptor. Este campo es opcional. Si no se envia se cobrará el monto asignado al Suscriptor ( Monto personalizado por Suscriptor ) o el monto de la Suscripción.
